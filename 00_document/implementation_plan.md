# Godot ホラーゲーム実装計画書 (α版)

## 1. プロジェクトセットアップ
- [ ] Godot 4.x プロジェクトの新規作成
- [ ] ディレクトリ構造の作成
    - `assets/`: 画像、音声、フォント
    - `data/`: シナリオJSON
    - `scenes/`: ゲームシーン、UI、プレハブ
    - `scripts/`: GDScriptファイル
- [ ] プロジェクト設定
    - 解像度設定 (1920x1080)
    - 入力マップ設定 (決定、キャンセル、QTEアクション)

## 2. コアシステム実装
### 2.1 ゲームマネージャー (GameManager)
- [ ] グローバル変数の管理 (`fear`, `kizuna`, `kegare`, `flags`)
- [ ] 状態管理 (タイトル、ゲーム中、ポーズ、QTE中)
- [ ] セーブ/ロード機能の実装 (`user://` ディレクトリへの読み書き)

### 2.2 シナリオマネージャー (ScenarioManager) & 演出システム
- [ ] **拡張JSONパーサー**: テキストと演出コマンドを時系列順に処理するロジック
- [ ] **コマンド実行クラス**: `CommandExecutor`
    - 各種演出 (Shake, Fade, Audio, Tween) を実行するモジュール
- [ ] シーン遷移ロジック
- [ ] イベント処理

### 2.3 オーディオマネージャー (AudioManager)
- [ ] BGMのクロスフェード機能
- [ ] SE再生機能 (位置、ピッチ、ボリューム変動機能付き)
- [ ] 音量設定の適用

## 3. プレビュー・効率化ツール実装 (優先度: 高)
シナリオ演出の試行錯誤を高速化するための機能。
- [ ] **ホットリロードシステム**:
    - `user://` または `res://` の特定フォルダのJSON変更を検知 (またはリロードボタン実装)
    - 現在のステートを保持したまま、シナリオデータを再読み込みする
- [ ] **Debug UI (Scenario Director)**:
    - オーバーレイUIの実装 (F1キーなどでトグル)
    - [ ] シーンID指定ジャンプ
    - [ ] 指定行へのジャンプ
    - [ ] 変数 (`fear`, `kizuna`) のリアルタイム編集
- [ ] **QTE Tuner (QTE調整ツール)**:
    - [ ] QTEパラメータ (Duration, Size, Text) をスライダー/入力欄で調整できるUI
    - [ ] "Test Run" ボタンで即座にその設定でQTEを発動させる機能
    - [ ] 調整結果のクリップボードコピー機能

## 6. 検証計画 (Verification Plan)
### 6.1 自動テスト
- [ ] `ScenarioManager` の単体テスト (JSON読み込み確認)
- [ ] `GameManager` のパラメータ計算ロジックのテスト

### 6.2 手動検証
- [ ] **ホットリロード動作確認**: JSON編集後、ゲーム内で即座に反映されるか。
- [ ] **演出コマンド動作確認**: 指定通りの演出 (音、揺れ、フェード) が再生されるか。
- [ ] **シナリオ進行確認**: 最初から最後までエラーなく進行すること。
- [ ] **選択肢分岐確認**: 選択肢によってパラメータが変化し、ルートが変わること。
- [ ] **QTE動作確認**: 成功/失敗が正しく判定され、結果が反映されること。
- [ ] **セーブ/ロード確認**: 途中から再開できること。
- [ ] **エンディング確認**: 条件を満たしたエンディングに到達すること。
